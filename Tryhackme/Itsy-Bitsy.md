# CTF Write-Up: Itsy-Bitsy

## Challenge Overview

* Platform: TryHackMe
* Challenge: Itsy-Bitsy
* Category: SOC Investigation / Threat Hunting
* Objective: Investigate HTTP connection logs in Kibana to identify indicators of compromise, determine the attacker's C2 infrastructure, and recover the flag.

---

## Scenario

During routine SOC monitoring, an IDS generated an alert indicating possible Command and Control (C2) communication originating from a user named **Browne** in the HR department. A week's worth of HTTP connection logs had been ingested into the `connection_logs` index in Kibana for investigation.

The goal was to analyze these logs and answer a series of forensic questions.

---

## Question 1: How many events were returned for the month of March 2022?

The time filter in Kibana was adjusted to include logs from **1 March 2022** to **31 March 2022**.

**Answer:**

```text
1482
```

---

## Question 2: What is the IP associated with the suspected user?

The logs were filtered to identify activity generated by the suspected user. While reviewing the HTTP requests, one host consistently used an unusual **User-Agent** string.

The associated IP address was:

```text
192.166.65.54
```

The suspicious indicator was the use of the **bitsadmin** User-Agent, which is commonly abused by malware to download payloads from remote servers.

---

## Question 3: What legitimate Windows binary was used to download the malicious file?

Reviewing the HTTP request headers revealed the following User-Agent:

```text
bitsadmin
```

`bitsadmin` is a legitimate Windows utility used for Background Intelligent Transfer Service (BITS), but it is frequently abused by attackers to download malware while blending in with normal system activity.

**Answer:**

```text
bitsadmin
```

---

## Question 4: Which file-sharing site was used as the Command and Control (C2) server?

Further analysis of the HTTP Host field showed repeated connections to:

```text
pastebin.com
```

Pastebin is a legitimate text-sharing website, but it is also commonly abused by malware authors to host commands, payloads, or configuration files.

**Answer:**

```text
pastebin.com
```

---

## Question 5: What is the full C2 URL?

Inspecting the request URI revealed the complete URL contacted by the infected machine:

```text
pastebin.com/yTg0Ah6a
```

---

## Question 6: What file was accessed from the C2 server?

Opening the Pastebin URL revealed that the malware attempted to access the following file:

```text
secret.txt
```

---

## Flag

```text
THM{SECRET__CODE}
```

---

## Key Takeaways

* Kibana is an effective platform for investigating network logs and identifying malicious activity.
* Unusual User-Agent strings can quickly reveal malware using legitimate Windows utilities.
* Tools such as **bitsadmin** are frequently abused by attackers as Living-off-the-Land Binaries (LOLBins).
* Legitimate services like Pastebin can be misused as Command and Control infrastructure.
* Correlating timestamps, IP addresses, User-Agent strings, and HTTP requests is essential during SOC investigations.
